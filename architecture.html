تصور معماري (مبسّط)

هذا النموذج تفاعلي واجهة أمامية. للتكامل الحقيقي مع محركات التعلم الآلي والمنصات (مثل Allam، Triton، Milvus) تحتاجون نقطة نهاية (API) على خادم (مثلاً Python/Flask أو Node.js) مزود بــ GPU، ومخزن بيانات مُؤمّن.

  1. الواجهة (Front-end) — صفحات HTML/CSS/JS (موجودة هنا).
  2. API Gateway — نقطة استقبال البلاغات وفحص الروابط.
  3. محرك الفحص — Allam/Triton لتشغيل نماذج الكشف، Milvus لقاعدة المعرفات (embeddings) إن لزم.
  4. قاعدة بيانات احتياطية — PostgreSQL / MongoDB لسجلات البلاغات.
  5. إشعارات — بوابات SMS والبريد والدفع داخل أبشر.

للعرض التجريبي: الواجهة هنا تحاكي سلوك الفحص والبلاغات بدون باك-إند. عند الانتقال للإنتاج نوصي بنشر نموذج Allam على Triton Server مع قاعدة Milvus للبحث التشابهي، واستخدام بنية GPU Node.
